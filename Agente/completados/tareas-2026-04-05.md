## 044A-44 — Migración de todos los repositorios a query_as!/query!/query_scalar! (compilación verificada)

- **Que:** Migrados los 8 archivos de repositorios restantes de `sqlx::query_as::<_, T>(...)` (runtime) a `sqlx::query_as!(T, ...)` (macro, verificación en compilación). También migrados `sqlx::query()` a `sqlx::query!()`, `sqlx::query_scalar::<_, T>()` a `sqlx::query_scalar!()`, y `sqlx::query_as::<_, (T,)>()` a `sqlx::query_scalar!()`. Para modelos con enums PostgreSQL (OrderStatus, PaymentMode, PhaseStatus, DelegationStatus, RefundStatus, UserRole) se utilizan type overrides: `column as "column: EnumType"` en SELECTs y `value as EnumType` en bindings. Las queries con `RETURNING *` se reemplazaron por listas explícitas de columnas cuando el modelo contiene enums. Cache SQLx offline regenerado con `cargo sqlx prepare`.
- **Archivos:** src/repositories/user.rs, src/repositories/payment.rs, src/repositories/order.rs, src/repositories/note.rs, src/repositories/chat.rs, src/repositories/delegation.rs, src/repositories/deliverable.rs, src/repositories/refund.rs, .sqlx/ (105 archivos de cache)
- **Gotchas:** No se pueden usar constantes string en macros proc (queries deben ser literales). `RETURNING *` no funciona con `query_as!` cuando hay enums — hay que listar columnas. Para `COUNT(*)` usar `as "count!: i64"` type override. Para `COALESCE` en JOINs usar `"field!: Type"` para indicar NOT NULL.
- **Sentinel:** No aplica

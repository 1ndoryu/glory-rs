## 044A-43 — Cambiar foto de perfil (avatar upload)

- **Que:** Implementado desde cero: migración DB (avatar_url, display_name en users), endpoint REST (GET /api/profile, POST /api/profile/avatar multipart), servido estático de uploads/, frontend conectado con input file oculto activado por botón "Cambiar foto". Las imágenes se guardan en uploads/avatars/{user_id}.{ext} con validación MIME (jpeg/png/webp/gif) y límite 2MB.
- **Archivos:** migrations/20260406000000_user_profile.up.sql, migrations/20260406000000_user_profile.down.sql, src/models/user.rs, src/repositories/user.rs, src/handlers/profile.rs (nuevo), src/handlers/mod.rs, frontend/src/api/profile.ts (nuevo), frontend/src/hooks/usePerfil.ts, frontend/src/components/panel/SeccionPerfil.tsx, frontend/src/components/panel/SeccionPerfil.css
- **Gotchas:** `image/*` dentro de comentarios Rust `/* */` causa nested block comment error. utoipa no soporta `request_body(content_type = "multipart/form-data")` — omitir como en deliverables. `crate::lib::AppState` no funciona — usar `crate::AppState`.

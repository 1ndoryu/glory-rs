# Tareas completadas — 2026-03-25

## 253A-1: Crear template glory-rs (Rust + React + OpenAPI + Codegen + Clippy paranoia)

**Qué se hizo:**
- Creado proyecto completo desde cero con Axum, SQLx, utoipa, serde, tracing, thiserror, jsonwebtoken, argon2, validator, tower-http, dotenvy
- Backend con arquitectura en capas: handlers/ → services/ → repositories/ → models/ → errors/
- Ejemplo funcional: CRUD de notas con autenticación JWT (register, login, CRUD)
- Middleware de autenticación JWT como Axum extractor (AuthUser)
- Migraciones SQLx reversibles (up/down) para users y notes
- OpenAPI generado desde código con utoipa, Swagger UI integrado
- Clippy nivel paranoia: `clippy::all = deny`, `clippy::pedantic = warn`
- Frontend React + TypeScript + Vite con React Query, Zustand, Orval para codegen
- Proxy configurado en Vite para requests API al backend
- CSS con variables y nombres en español
- .gitignore, .env.example, README.md completo con instrucciones

**Decisiones:**
- Orval reemplaza a openapi-typescript-codegen (más moderno, genera React Query hooks)
- No se incluyó `config` crate — dotenvy + env vars es el estándar 12-factor para containers
- Se usa `sqlx::query_as` (no macro) para que el template compile sin DB conectada
- `#[async_trait]` necesario para `FromRequestParts` en axum 0.7 (axum-core 0.4.5)

**Validaciones:**
- `cargo check` ✓
- `cargo clippy` ✓ (0 warnings)
- `cargo fmt --check` ✓
- `cargo test` ✓ (0 tests, compilación exitosa)

**Nota:** Test funcional omitido — psql no en PATH y DB glory_db no creada aún. El usuario debe crear la DB y configurar .env para probar los endpoints.

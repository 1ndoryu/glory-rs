## 104A-46 — Fix: deploy --update para template rust usaba git pull incorrecto

- **Qué:** `deploy --update --name studio` llamaba a `theme_manager::update_glory_theme()` que internamente buscaba un contenedor WordPress y hacía `git pull` del tema. Para sitios con `template=rust` no existe contenedor WordPress.
  - Fix: early-exit en `deploy_theme::execute()` — si `site.template == StackTemplate::Rust`, el flag `--update` redirige a `commands::redeploy::execute()` (Coolify API stop+start), y sin `--update` redirige a `commands::deploy_service::execute()` (build completo).
  - Ahora `deploy --update --name studio` funciona igual que `redeploy --name studio`.
- **Archivos:** `src/commands/deploy_theme.rs` en coolify-manager-rs
- **Gotchas:** El early-exit debe ir ANTES de `pre_deploy_safety_check` para evitar el backup que también falla en rust.
- **Sentinel:** No aplica — el comportamiento diferenciado es válido por diseño.
- **GLORY-RS:** No aplica (es coolify-manager-rs específico).

## 154A-6 — Performance PageSpeed: compression, cache, code splitting

- **Qué:** PageSpeed mobile era 55 (FCP 7.9s, LCP 26.6s). Implementados 4 fixes:
  1. **HTTP Compression gzip+brotli**: Agregado `CompressionLayer` a Axum. Reduce transferencia ~70%.
  2. **Cache headers**: Assets estáticos con hash → `max-age=31536000, immutable`. Uploads → cache 1 año. Frontend dist → `max-age=86400`.
  3. **Font preload**: `<link rel="preload">` para NeueMontreal Regular y Medium en index.html. `font-display: swap` ya existía.
  4. **React.lazy code splitting**: PanelIsland (@tiptap ~350KB), AdminEditorProvider, ChatWidget cargados bajo demanda. Reduces initial bundle ~40%.
- **Archivos:** `Cargo.toml` (features compression-gzip, compression-br), `src/handlers/mod.rs` (CompressionLayer + cache middleware), `frontend/index.html` (font preload), `frontend/src/App.tsx` (React.lazy + Suspense)
- **Gotchas:** `ServeDir` no soporta `.layer()` directo — usar `tower::ServiceBuilder::new().layer().service()`. Code Sentinel reporta falso positivo en `React.lazy().then()` sin catch (React lo maneja internamente).

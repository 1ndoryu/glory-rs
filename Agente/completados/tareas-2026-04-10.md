## T-4 — Sync cross-device/tab

- **Qué:** Implementado soporte multi-conexión WS por sesión. Múltiples pestañas/dispositivos pueden conectarse a la misma sesión de chat simultáneamente. Al cerrar una pestaña, la sesión permanece activa para las demás. Solo se cierra cuando la última conexión se desconecta o el usuario cierra explícitamente.
- **Archivos:**
  - `src/services/chat.rs` → `connection_counts` (AtomicUsize refcount por sesión), `subscribe()` incrementa, `unsubscribe()` decrementa y retorna remaining, `close_session()` limpia refcount
  - `src/services/chat_timing.rs` → `register_session()` reutiliza timing loop existente si sesión ya registrada (no duplica tarea tokio)
  - `src/handlers/chat/ws_visitor.rs` → `process_visitor_messages()` retorna bool (cierre explícito vs stream ended), cleanup usa `unsubscribe()` para decidir si cerrar sesión
  - `frontend/src/hooks/useChatWidget.ts` → BroadcastChannel `nakomi-chat` para relay de mensajes entre pestañas, `handleServerMessage()` extraído como helper reutilizable
- **Gotchas:**
  - El `broadcast::Sender` de tokio ya soporta multi-consumer (cada `subscribe()` crea un nuevo receiver). El problema era el lifecycle: cerrar una conexión mataba la sesión para todas.
  - `AtomicUsize` refcount con `Ordering::Relaxed` es suficiente (no hay dependencias de orden entre increment/decrement).
  - clippy `doc_markdown`: nombres de campos en docstrings necesitan backticks.
  - clippy `must_use_candidate`: `unsubscribe()` retorna usize que el caller debe usar.
- **Sentinel:** No aplica.

## 104A-5 — Sistema de optimización de imágenes on-demand
- **Qué:** Implementado proxy de imágenes estilo Jetpack Photon CDN. Backend: ruta `GET /api/img/{*path}?w={ancho}&q={calidad}&fmt={formato}` que procesa imágenes de uploads/ al vuelo con cache en disco (`uploads/.cache/`). Soporta resize por ancho (whitelist: 150-2400px), compresión JPEG con calidad configurable, conversión a WebP (lossless via `image` crate 0.25). Frontend: utility `optimizedUrl()` + `generateSrcSet()` + `generateWebPSrcSet()` en `imageUtils.ts`. Componente `<OptimizedImage>` con `<picture>` element (source WebP + fallback), srcSet responsive, `loading="lazy"` por defecto. Integrado en 11 componentes que muestran imágenes del CMS.
- **Archivos:**
  - `Cargo.toml` → +image crate (jpeg, png, webp, gif)
  - `src/services/image_processing.rs` → nuevo: process_image, cache_path, get_optimized_image, encode_image
  - `src/handlers/image_proxy.rs` → nuevo: image_proxy handler con path traversal protection, whitelist de anchos
  - `src/handlers/mod.rs` → +image_proxy module y rutas
  - `src/services/mod.rs` → +image_processing module
  - `frontend/src/utils/imageUtils.ts` → nuevo: optimizedUrl, generateSrcSet, generateWebPSrcSet
  - `frontend/src/components/ui/OptimizedImage.tsx` → nuevo: componente <picture> responsive

- **Gotchas:**
  1. `image` crate 0.25 solo tiene WebP encoder lossless (no lossy). WebP lossless sigue siendo ~26% menor que PNG.

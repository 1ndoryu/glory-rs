## 165A-3 — Compose SSH parseable para provisioning de hosting

- **Que:** El `500` restante en producción no venía del contrato HTTP sino del compose generado para el sidecar SSH/SFTP: Coolify no podía parsear el `dockerfile_inline` y abortaba el alta cerca del `FROM lscr.io/linuxserver/openssh-server...`. Se reescribieron los builders de SSH con literales YAML claros, se eliminó basura de diff que había quedado en el compose estático, y se añadieron pruebas que parsean el YAML generado tanto para hosting WordPress como para hosting normal.

## 303A-8 — Deploy zero-downtime via SSH
- **Qué:** Reescrito `scripts/deploy.ps1` para eliminar el uso de Coolify restart (que mata contenedores antes de construir, causando 404). Nuevo flujo: PATCH compose a Coolify → `docker compose build` via SSH (viejo sigue sirviendo) → `docker compose up -d --no-build` (swap instantáneo) → conectar Traefik → health check.
- **Archivos:** `scripts/deploy.ps1`, `Agente/lecciones/lecciones-aprendidas.md`
- **Gotchas:**
  - Coolify restart API hace `docker compose up -d --force-recreate --build` que mata el viejo contenedor antes de que el build termine → 404 durante 5-7 min.
  - A veces Coolify ni siquiera inicia el build después del restart, dejando el servicio muerto.
  - El fix anterior de SQLx offline cache (`.sqlx/`) era necesario para que el build compilara. SIEMPRE ejecutar `cargo sqlx prepare` tras cambiar queries.
  - Traefik necesita estar en la red del servicio — se conecta automáticamente ahora.
- **Sentinel:** Posible regla: detectar si `.sqlx/` tiene archivos stale cuando queries cambian (complejo, bajo ROI por ahora).

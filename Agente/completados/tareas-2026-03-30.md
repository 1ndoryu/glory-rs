- **Archivos:**

  - `src/models/integracion_marketing.rs` — nuevo campo + `meta_templates_configurado()` + `meta_configurado()` actualizado

  - `src/services/campana.rs` — `enviar_por_canales` retorna contadores, actualiza destinatarios
  - `src/services/recordatorio.rs` — compone número E.164 con prefijo

- **Gotchas:**
  - La API de mensajes de Meta usa `Phone-Number-ID` (POST /{Phone-Number-ID}/messages), NO el WABA ID. El WABA ID solo sirve para gestión de templates.
  - Meta API actual es v23.0, teníamos v21.0.
  - Los recordatorios usaban `r.telefono` de reservas que NO tiene prefijo de país. Se necesita JOIN con `clientes.prefijo_telefono`.
  - `enviar_por_canales` era fire-and-forget (no retornaba nada). Los contadores `total_enviados`/`total_fallidos` quedaban siempre en 0.
  - Los destinatarios individuales de campañas quedaban eternamente en estado "pendiente".
- **Sentinel:** La regla de "no usar stubs en producción" podría detectar patrones como `stub_` o `integración pendiente` en código, pero es bajo valor operacional.

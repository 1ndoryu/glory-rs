# Tareas completadas — 2026-03-31

## 313A-4 — Reducir falsos positivos en indicadores off-screen
- **Qué:** Ajustada la lógica de `OffScreenIndicators` para que no muestre flechas cuando el viewport aún no está medido o cuando una mesa queda apenas fuera por decimales/tolerancia de borde. Se añadió `INDICATOR_EPSILON = 6` px y guard clause para `viewportWidth/viewportHeight <= 0`.
- **Archivos:**
  - `frontend/src/componentes/plano-sala/OffScreenIndicators.tsx`
- **Gotchas:** Con pan basado en `transform` y zoom decimal, los límites visibles pueden quedar en valores no enteros. Sin tolerancia, una mesa a fracciones de pixel del borde activa el indicador aunque visualmente siga dentro.
- **Sentinel:** No aplica.

## 313A-5 — Fix minimap proporciones + viewport rect invisible + resize handles mesas
- **Qué:** 4 correcciones relacionadas al plano de sala y minimap:
  1. **contentBounds inflado**: se eliminó `canvasHeight` del cálculo de `contentBounds.h` en PlanoSala y PlanoOcupacion. canvasHeight es la altura del viewport, no del contenido. Incluirlo hacía que el minimap tuviera proporciones incorrectas y el viewport rect cubriera 100% del alto (invisible).
  2. **Viewport rect invisible en minimap**: se rehízo el dibujado del minimap — ahora oscurece el área fuera del viewport y dibuja el viewport rect con mayor opacidad/grosor. Técnica de "darkening overlay" con clearRect + redraw selectivo.
  3. **Resize handles de mesas desaparecidos**: se reemplazaron `<Button>` de shadcn por `<div>` planos. El Button inyectaba clases Tailwind (`size-6`, `inline-flex`, `border-transparent`) que conflictuaban con el CSS custom de `.planoMesaResizeHandle` (14px circles), haciendo los handles invisibles o mal posicionados.
  4. **Minimap siempre visible**: ahora se muestra siempre que haya mesas con contenido real, usando `effectiveWidth/Height = max(content, viewport)` para proporciones correctas incluso cuando no hay scroll.
- **Archivos:**
  - `frontend/src/componentes/PlanoSala.tsx` (contentBounds fix)
  - `frontend/src/componentes/PlanoOcupacion.tsx` (contentBounds fix)
  - `frontend/src/componentes/plano-sala/CanvasMinimap.tsx` (viewport rect + visibility)
  - `frontend/src/componentes/plano-sala/MesaDraggable.tsx` (resize handles)
- **Gotchas:** En Tailwind v4, las clases utility se aplican en `@layer utilities` que en teoría pierde ante CSS sin layer. Pero la combinación de `bg-clip-padding`, `size-6`, `border-transparent` etc. del Button de shadcn creaba conflictos sutiles. Usar `<div>` planos elimina toda interferencia.
- **Sentinel:** No aplica.

## 064A-3 — Filtros por columna en tablas
- **Qué:** Componente `ColumnFilterHeader` reutilizable con Popover + checkboxes para filtrado multi-valor. Implementado en Ventas (Turno, Canal, Método), Gastos (Tipo doc, Método) y Reservas (Estado). Backend acepta params comma-separated y filtra via `string_to_array()` en PostgreSQL.
- **Archivos:** `frontend/src/components/column-filter-header.tsx` (nuevo), 3 hooks (`useListaVentas`, `useListaGastos`, `useVistaReservas`), 3 listas (`ListaVentas`, `ListaGastos`, `ListaReservas`), modelos/repos/servicios/handlers backend para ventas y gastos, repositorio reservas (estado multi-valor).
- **Gotchas:** Reserva COUNT query tuvo que convertirse de `query_scalar!` (compile-time) a runtime `query_scalar::<_, Option<i64>>` porque `string_to_array` no es soportado por el macro de SQLx offline. Los doc comments en modelos con valores separados por coma necesitan backticks para pasar clippy::doc_markdown.
- **Sentinel:** No aplica.

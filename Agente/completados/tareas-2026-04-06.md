- **Gotchas:** El token API usa `#[serde(skip_serializing)]` para no exponerlo en respuestas GET, similar a groq_api_key. - **Pendiente de prueba real:** No se puede probar contra API real hasta que el cliente proporcione su token. Tests unitarios cubren mapeo de datos.

## 064A-7 — Prevención de duplicados + idempotencia
- **Qué:** Mutex por venta (`LazyLock<Mutex<HashMap<Uuid, Arc<TokioMutex>>>>`) evita syncs concurrentes. Guard en create: re-lee venta de BD, si `haddock_synced=true` → skip. Parámetro `is_update` distingue create/update — updates siempre re-envían, creates solo si no synced.
- **Archivos:** `src/services/haddock.rs` (mutex estático, guard, cleanup_lock, `is_update` en sync_order/sync_order_to_url), `src/services/venta.rs` (spawn con is_update), `src/services/reserva.rs` (spawn con is_update=false)
- **Gotchas:** Clippy exige `let...else` en vez de `match` para `try_lock`. `cleanup_lock` limpia el HashMap cuando `Arc::strong_count <= 2` para prevenir memory leak.
- **Sentinel:** No aplica.

## 024A-3 — Filtro "día completo" no mostraba todas las reservas (stale cache)
- **Qué:** Reemplazar `refetch()` (que solo refrescaba la query actual) con `queryClient.invalidateQueries()` usando query keys base sin params, para invalidar TODAS las variantes de queries de reservas y ocupación tras cualquier mutación.
- **Archivos:**
  - `frontend/src/hooks/useVistaReservas.ts` (invalidarReservas() centralizado + imports getListarReservasQueryKey)
- **Gotchas:** React Query con staleTime global de 5 minutos. `refetch()` solo ejecuta la query con los params actuales. Al cambiar de filtro (ej: cena → día completo), la nueva combinación de params tenía su propia cache entry que seguía stale. La solución correcta es invalidar con el query key base `['/api/reservas']` que matchea todas las variantes.
- **Sentinel:** No aplica

## 024A-4 — Fix scheduler error: column 13 email_cliente nullable
- **Qué:** El scheduler de recordatorios fallaba con "unexpected null; try decoding as an 'Option'" en la columna 13 (email_cliente). El campo venía de `LEFT JOIN clientes c ON c.id = r.cliente_id` — cuando no existe cliente vinculado, `c.email` es NULL. SQLx offline no infiere nullable de LEFT JOINs, así que el cache `.sqlx` marcaba `nullable: false`.
- **Archivos:**
  - `.sqlx/query-a8062f8a829190871e768ba44c6f7ae1f6b8310224b8aba00cddc8cfc4857cc2.json` (nullable[13] cambiado de false a true)
  - `migrations/20260402000000_campana_plantilla_whatsapp.up.sql` (hecho idempotente con IF NOT EXISTS para evitar fallo en deploy cuando la migración ya fue aplicada manualmente)
- **Gotchas:** El struct `ReservaPendienteRecordatorio` ya tenía `email_cliente: Option<String>` pero el cache `.sqlx` no concordaba. SQLx confía en el cache para la deserialización en runtime. También descubrimos que el deploy anterior falló porque la migración plantilla_whatsapp ya existía en la DB (aplicada manualmente vía docker exec) pero no en _sqlx_migrations.
- **Sentinel:** Posible regla: detectar LEFT JOINs en queries sqlx donde el campo seleccionado no tiene override `?` y el struct lo declara como Option.

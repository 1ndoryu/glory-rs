## 294A-1 - Automatizacion real de extracción/scraping y reporte interno
- **Que:** Se corrigió la causa raíz de los lotes en error: los procesos Python sí se ejecutaban, pero no recibían URL interna del backend para reportar cierre. Rust ahora deriva `BACKEND_URL` desde `HOST`/`PORT`, pasa `SCRAPER_SECRET`/`KAMPLES_CRON_SECRET` al hijo, crea lotes antes de lanzar procesos, levanta un worker automático real y persiste reportes de extracción/scraping en `lotes_procesamiento`.
- **Archivos:** `src/services/admin_processes.rs`, `src/services/admin_automation/mod.rs`, `src/repositories/admin_automation.rs`, `src/repositories/mod.rs`, `src/handlers/scraper_admin.rs`, `src/workers/automation_worker.rs`, `src/workers/mod.rs`, `src/main.rs`, `clients/kamples-scraper/scripts/cron_runner.py`, `clients/kamples-scraper/extractor/pipeline.py`, `frontend/src/legacy/hooks/useAutomationProcessCard.ts`, `roadmap.md`, `Agente/documentacion/backend/admin-automatizacion-samples-2026-04-28.md`.
- **Gotchas:** La captura del panel parecia decir que WhoSampled fallaba todo, pero el log mostraba `No se puede reportar lote scraper — URL/secret no configurados`: el scraper terminaba, pero el lote quedaba sin cierre y el reconciliador lo marcaba como error. Ahora `termino sin reportar cierre` se trata como fallo de telemetría y no infla fallos consecutivos del scraper/extractor.
- **Validacion:** `cargo fmt`, `cargo check`, `cargo clippy --all-targets -- -D warnings`, `cargo test` (181 tests), `npm --prefix frontend run type-check`, `npm --prefix frontend run build`, `clients\\kamples-scraper\\.venv\\Scripts\\python.exe -m py_compile clients\\kamples-scraper\\scripts\\cron_runner.py clients\\kamples-scraper\\extractor\\pipeline.py`, `cargo run -- --emit-openapi openapi.json` con `CARGO_TARGET_DIR=C:\\tmp\\glory-openapi-target`, `npm run codegen`, `npm run self-check -- -TareaId 294A-1`.
- **Sentinel:** No requiere regla nueva. Se redujo `spawn_python()` y se justificó `limite-lineas` en `admin_processes.rs` porque es un gestor legacy central de procesos, locks, cookies y entorno Python.
- **GLORY-RS:** No aplica. El worker y las rutas dependen de procesos, tablas y scripts específicos de Kamples.

## 044A-1 — Deploy refactorizado a coolify-manager-rs deploy-service
- **Que:** Reemplazado el deploy.ps1 hardcodeado (SSH directo, IPs/UUIDs fijos) con un wrapper que invoca `coolify-manager-rs deploy-service`. El nuevo comando en coolify-manager-rs implementa el flujo completo de deploy zero-downtime: sync compose con Coolify API → verificar postgres → build imagen → swap contenedor → conectar Traefik → health check → seed opcional.
- **Archivos modificados:**
  - `scripts/deploy.ps1` — Reescrito como wrapper: localiza binario, config, y reenvía flags
  - `scripts/update-coolify-compose.ps1` — Marcado como deprecated (compose sync ahora vive en deploy-service)
  - `roadmap.md` — Actualizado header de deploy
  - **coolify-manager-rs:**
    - `src/commands/deploy_service.rs` — Nuevo: comando completo de deploy zero-downtime
    - `src/commands/mod.rs` — Agregado módulo deploy_service
    - `src/cli/mod.rs` — Agregado subcomando DeployService con flags --skip-build, --seed, --skip-compose-sync
    - `src/domain/mod.rs` — Agregado campo `repo_url: Option<String>` a SiteConfig
    - `src/infra/template_engine.rs` — `rust_vars()` ya no incluye DB_PASSWORD/JWT_SECRET (se dejan como ${} para Coolify)
    - `src/services/health_manager.rs` — Fix: Rust apps usan HTTP check como app_ok (debian slim no tiene pgrep)
    - `src/services/migration_manager.rs` — Actualizado caller de rust_vars
    - `src/commands/new_site.rs` — Actualizado caller de rust_vars + nuevo campo repo_url
    - `templates/rust-stack.yaml` — Actualizado a match real: submodules, mount caches, seed binary, ca-certificates
    - `config/settings.json` — Actualizado dominio glory-rest + repoUrl (no tracked, contiene secrets)
- **Gotchas:**
  - Health check interno fallaba porque `/app/app` no existe en nuestro container (el binario es `/app/glory-backend`). Simplificado: para Rust apps, si HTTP responde 200, app_ok=true.
  - `pgrep -f '/app/'` no disponible en debian:bookworm-slim — no hay procps.
  - Path resolution del config: binary está en `target/release/`, hay que subir 3 niveles para llegar al root de coolify-manager-rs.
  - `rust_vars()` ya no recibe DB_PASSWORD/JWT_SECRET — esos se dejan como `${VAR}` en el template para que Coolify los resuelva. Solo DOMAIN, GLORY_BRANCH, REPO_URL son template vars.
- **Sentinel:** No aplica (cambio cross-project, no frontend/backend específico)
- **Testeado:** `.\scripts\deploy.ps1 -SkipBuild -SkipComposeSync` → Deploy exitoso, health check OK (status 200)

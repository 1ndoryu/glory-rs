## 264A-1 - Autolimpieza del target de Cargo en npm run dev

- **Que:** Se reforzo la limpieza automatica del target de Cargo usado por `npm run dev`. La limpieza segura ahora poda `incremental/` cuando `C:/tmp/glory-target` supera 4096 MB. Se agrego un watcher (`scripts/watch-cargo-target.ps1`) que vigila el tamaño durante sesiones largas y ejecuta limpieza `-Hard` si no hay `rustc` compilando. `package.json` ahora ejecuta limpieza previa al arranque y el watcher en paralelo con backend/frontend. `README.md` y la documentacion del agente registran el flujo.
- **Archivos:** `scripts/clean-cargo-target.ps1`, `scripts/watch-cargo-target.ps1`, `package.json`, `README.md`, `Agente/documentacion/frontend/dev-cargo-target-autoclean-2026-04-26.md`
- **Gotchas:** `-Hard` ya no mata `glory-backend`; solo `-Aggressive` lo hace. Eso permite autolimpieza sin tirar abajo el servidor dev. En esta maquina `C:/tmp/glory-target` no existia al validar, asi que los scripts se probaron sobre la ruta vacia y el wiring npm.
- **Sentinel:** No aplica.
- **GLORY-RS:** Aplica parcialmente. La estrategia de autolimpieza de `cargo target` es reutilizable para otros proyectos Rust del framework; por ahora queda implementada en este template y puede extraerse a `glory-rs` si se replica el mismo flujo de scripts.

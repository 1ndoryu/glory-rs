## 134A-21 — Fix flicker drag mesa: eliminar dnd-kit, pointer events nativos
- **Qué:** El arrastre de mesas mostraba un "doble-delta" visual: al soltar, la mesa aparecía en `newPos*zoom + delta` por un frame antes de estabilizarse. Causa raíz: dnd-kit aplica `transform: translate(dx, dy)` durante el drag + mantiene `left/top` en la posición original. Al soltar, el reset del transform de dnd-kit y la actualización de `posicionesLocales` no ocurren en el mismo ciclo de render de React → un frame con posición doble. Fix definitivo: eliminar dnd-kit de mesa drag completamente y usar el patrón ParedDraggable (refs + forceUpdate + pointer events nativos, sin CSS transform).
- **Archivos:**
  - `frontend/src/componentes/plano-sala/MesaDraggable.tsx` (reescrito con pointer events)
  - `frontend/src/componentes/plano-sala/usePlanoSala.ts` (handleDragStart/End → handleMoverMesa, quitado arrastrando + canvasRef)
  - `frontend/src/componentes/PlanoSala.tsx` (quitado DndContext + sensors + imports dnd-kit)
- **Gotchas:**
  - `e.preventDefault()` en onPointerDown suprime el click del browser — sin esto onClick se dispara doble (pointer events + click event)
  - Los resize handles usan `onMouseDown` (useMesaResize); agregar `onPointerDown={(e) => e.stopPropagation()}` a los handles evita que el drag de la mesa se active al redimensionar
  - `setPointerCapture(e.pointerId)` es obligatorio para trackear el drag fuera del elemento
  - `canvasRef` en usePlanoSala era exclusivo de handleDragEnd (para `clientWidth / zoom`). Al eliminarlo, se quitó también el parámetro y el import de RefObject
  - CLICK_THRESHOLD = 4px distingue click de drag para que onClick siga funcionando normalmente
- **Sentinel:** No aplica

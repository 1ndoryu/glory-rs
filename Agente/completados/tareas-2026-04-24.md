## 124A-IMG — Imágenes carrusel: fetchPriority LCP, sizes correctas, calidad mejorada

- **Qué:** (1) `OptimizedImage` gana prop `fetchPriority?: 'high'|'low'|'auto'` propagada a `<img>`. (2) `CarruselShowcase`: primer item `loading="eager"` + `fetchPriority="high"` para LCP. El `sizes` se corrige de `"450px"` (demasiado pequeño para imágenes de altura fija 400-600px con ancho auto) a `"(min-width: 1440px) min(85vw, 900px), (min-width: 768px) min(80vw, 800px), 90vw"` — hint realista para imágenes de ratio 3:2 a 600px de alto. Quality sube de 80 a 85. (3) `SeccionShowcase`: sizes de `"(max-width: 768px) 100vw, 50vw"` a `"(min-width: 1024px) min(50vw, 640px), (min-width: 768px) 50vw, 100vw"` — mayor superposición con el srcSet del carrusel en el rango 640px para hits de cache compartidos.
- **Archivos:** `frontend/src/components/ui/OptimizedImage.tsx`, `frontend/src/components/home/CarruselShowcase.tsx`, `frontend/src/components/home/SeccionShowcase.tsx`
- **Gotchas:** `sizes` es un hint para el browser, no un valor exacto — el browser puede ignorarlo. El ResizeObserver mide el ancho real post-render y recalcula el srcSet, así que el sizes es crítico solo para la primera carga. `fetchPriority` es un atributo HTML5 estándar que React acepta como prop directamente.
- **Sentinel:** No aplica.
- **GLORY-RS:** `fetchPriority` en `OptimizedImage` es reutilizable globalmente — ya vive en el componente UI de glory-rs (la lógica de OptimizedImage está centralizada). Aplica.

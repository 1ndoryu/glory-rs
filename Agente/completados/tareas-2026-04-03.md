## 034A-4 — Evitar ventas duplicadas al re-guardar reserva completada
- **Que:** El handler `update` creaba una venta automática cada vez que se guardaba una reserva con `estado == "completada"`, sin verificar si ya estaba completada antes. Esto generaba ventas duplicadas si el usuario editaba cualquier campo de una reserva ya completada. Fix: se consulta el `estado` previo antes del update y solo se dispara auto-venta si realmente CAMBIÓ a "completada".
- **Archivos:** `src/services/reserva.rs`
- **Gotchas:** 1) `sqlx::query_scalar!` macro no infería el tipo nullable — se cambió a `Self::get()` para obtener el estado anterior completo. 2) Docker BuildKit `--mount=type=cache,target=/app/target` persistía binarios stale entre builds incluso con `--no-cache`. Se eliminó el mount cache de target del Dockerfile inline; la build toma ~8 min en vez de ~1 min pero garantiza binario fresco. 3) `docker builder prune -af` puede tardar >30s y debe completarse antes de reconstruir.
- **Sentinel:** No aplica

## 034A-2 — Fix clientes de reservas no aparecen en lista
- **Que:** `upsert_cliente` requería teléfono o email para crear un cliente; si la reserva se creaba sin teléfono (solo nombre), no se generaba registro de cliente. Se cambió la lógica: ahora solo necesita nombre para crear cliente. Si hay tel/email disponible, se usa para deduplicar. También se agregó upsert_cliente en el flujo de `update` cuando el estado cambia a "completada" y la reserva no tiene cliente vinculado (cubre reservas existentes creadas antes del fix).
- **Archivos:** `src/services/reserva.rs`
- **Gotchas:** Reservas anteriores sin cliente_id necesitan el check en update para vincularlas retroactivamente.
- **Sentinel:** No aplica

## 034A-2 — Fix clientes de reservas no aparecen en lista

- **Archivos:** `src/services/reserva.rs`
- **Gotchas:** Reservas anteriores sin cliente_id necesitan el check en update para vincularlas retroactivamente.

## 034A-4 — Evitar ventas duplicadas al re-guardar reserva completada
- **Que:** El handler `update` creaba una venta automática cada vez que se guardaba una reserva con `estado == "completada"`, sin verificar si ya estaba completada antes. Esto generaba ventas duplicadas si el usuario editaba cualquier campo de una reserva ya completada. Fix: se consulta el `estado` previo antes del update y solo se dispara auto-venta si realmente CAMBIÓ a "completada".
- **Archivos:** `src/services/reserva.rs`
- **Gotchas:** `sqlx::query_scalar!` macro no infería el tipo nullable de `estado`; se usó `sqlx::query_scalar` dinámico con type annotation `Option<Option<String>>` y `.flatten()`.
- **Sentinel:** No aplica

## 283A-6 — forwardRef en Button + SidebarMenuButton (React 18 compat) + fix dark mode toggle
- **Qué:** Los componentes shadcn generados con estilo `radix-vega` omiten `forwardRef` porque asumen React 19. En React 18 esto rompe el mecanismo `asChild` de Radix (Slot no puede pasar refs). Convertidos `Button` y `SidebarMenuButton` a `React.forwardRef`. Esto corrige: warnings de consola "Function components cannot be given refs" (task 39) y el toggle de dark mode que no funcionaba porque el DropdownMenu del ThemeToggle no recibía refs (task 41).
- **Archivos:** frontend/src/components/ui/button.tsx, frontend/src/components/ui/sidebar.tsx
- **Gotchas:** Todo componente shadcn que se use como hijo de un `asChild` trigger (Tooltip, DropdownMenu, etc.) necesita forwardRef en React 18. Si se genera un nuevo componente con `npx shadcn`, hay que verificar que use forwardRef.
- **Sentinel:** No aplica

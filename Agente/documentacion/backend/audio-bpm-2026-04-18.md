# Audio BPM — 2026-04-18

## Estado actual

- `174A-31`: existe `src/audio/bpm.rs` con detección de tempo basada en Symphonia, energía RMS, onset envelope y autocorrelación.
- El detector funciona tanto sobre muestras en memoria como sobre archivos reales (`detect_bpm_from_file`).
- El módulo no depende de FFmpeg ni de crates extra para análisis rítmico.

## Algoritmo implementado

- Ventana máxima de análisis: `30s`.
- Remuestreo determinista a `8 kHz` para abaratar el análisis y alinear el detector con el legado.
- Ventanas de energía:
  - `FRAME_MS = 50`
  - `HOP_MS = 25`
- Onset envelope:
  - derivada frame a frame sobre RMS
  - `half-wave rectification` (`max(0, diff)`)
  - normalización por pico máximo
- Autocorrelación:
  - rango de búsqueda `60..200 BPM`
  - selección del lag con mejor correlación media
  - `confidence = min(1, peak / (mean * 3))`

## API pública

- `detect_bpm(samples, sample_rate_hz) -> Option<BpmAnalysis>`
- `detect_bpm_from_file(path, format_hint) -> Result<Option<BpmAnalysis>, BpmError>`
- `BpmAnalysis` expone:
  - `bpm`
  - `confidence`
  - `analyzed_seconds`
  - `frame_count`

## Notas de migración

- El legado PHP exportaba PCM con FFmpeg antes de analizar. En Rust se elimina esa dependencia: Symphonia decodifica y el detector trabaja sobre muestras mono en memoria.
- El remuestreo actual es nearest-neighbor porque para BPM grueso da estabilidad suficiente con costo bajo. Si `174A-32` o `174A-33` necesitan más fidelidad espectral, convendrá extraer un resampler compartido de mayor calidad.
- La interfaz es agnóstica y candidata a `glory-rs`, pero se mantiene local hasta cerrar `audio_pipeline.rs` y confirmar el contrato que usarán worker y persistencia.
